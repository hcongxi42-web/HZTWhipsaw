**核心设计理念（P0 架构重构）**：

```
旧架构（线性加权平均）：              新架构（门控电源）：
  total = Σ(score × weight)           total = raw × gate × consistency
                                                 │       │         │
  问题：下跌股 strength=5              raw: 原始加权分（保留参考）

                                       consistency: 跨窗口一致性校验

```

### 3.1 洗盘质量 `score_washout_quality`（原始权重 15%）⚠️ 含 P0 门控

**度量什么**：下跌时是否缩量（洗盘特征 vs 出货特征）。

**P0 前置门控**：洗盘的前提是「有涨可洗」。如果 `strength_score < 20`，直接 return 0——缩量阴跌不是洗盘。

```python
def score_washout_quality(self, recent_days=15, strength_score=None):
    # P0门控：没有上涨前科的缩量下跌 = 阴跌，不是洗盘
    if strength_score is not None and strength_score < 20:
        return 0    # ← 堵死"缩量阴跌伪装洗盘"的套利路径
```

**子因子**：
| 子因子 | 权重 | 函数 | 关键参数 |
|--------|------|------|----------|
| 缩量程度 | 40% | sigmoid(avg_shrink, center=0.75, k=8) | `0.75` 中心点, `8` 陡峭度 |
| 缩量占比 | 35% | 线性: shrink_pct × 100 | `0.85` 缩量阈值 |
| 回撤深度 | 25% | bell(max_dd, mu=0.15, sigma=0.10) | `0.15` 最优回撤, `0.10` 宽度 |

**代码位置**：第 175-200 行


### 3.4 股票强度 `score_stock_strength`（原始权重 30%）⚠️ 门控电源核心

**度量什么**：洗盘前的上涨强度——好股票洗盘前应该有过像样的涨幅。

**核心设计思想**：股票强度不是和其他维度平起平坐的「30% 权重维度」，而是整个形态体系的「100% 电源开关」。强度不足时，直接 return 0，后续 Sigmoid 门控进一步压制，其他维度被整体关闭。

#### 趋势熔断流程（第 427-481 行）⚠️ P1 收紧

```
计算前期年化斜率 (annual_slope)
        │
  annual_slope < -5%  ？─── YES ──→ return 0    (硬闸1：明显下跌，直接出局 ⬆从5→0)
        │
  annual_slope < 0%   ？─── YES ──→ return 0~5  (硬闸2：微跌线性，⬆从5~15→0~5)
        │
  annual_slope < 3%   ？─── YES ──→ return 5    (硬闸3：横盘随机游走 ⬆从12→5)
  AND r² < 0.30
        │
  trend_score < 15    ？─── YES ──→ return trend_score (短路：不给量能/回调/相对优势救场)
        │                                    ⬆ 从 ×0.5→直接返回
        ▼
  正常计算四因子 + 加权平均
```

| 硬闸 | 旧返回值 | 新返回值 | 含义 |
|------|---------|---------|------|
| `annual_slope < -5%` | 5 | **0** | 下跌趋势直接出局 |
| `annual_slope < 0%` | 5~15 | **0~5** | 微跌几乎出局 |
| `annual_slope < 3% + r²<0.30` | 12 | **5** | 横盘随机游走 |
| 数据不足 | 5 | **0** | 无法评估=不通过 |
| `trend_score < 15` | return ×0.5 | **return trend_score** | 直接短路 |

#### 子因子 A：趋势强度 `_strength_trend`（第 484-516 行）

## 4. 门控架构（P0 核心重构）

> **旧架构**：`total = Σ(score × weight)` — 线性加权，下跌股可通过洗盘维度「跨维度套利」
> **新架构**：`total = raw × gate × consistency` — 股票强度是电源总闸，不是灯泡

### 4.1 四层门控流程

```
股票进入评分
    │
    ▼
┌─ 第一层：股票强度硬闸 ──────────────────────────┐
│ annual_slope < -5% → return 0 → 全维度直接归零    │
│ annual_slope < 0%  → return 0~5 → 几乎归零        │
│ annual_slope < 3% + r²<0.30 → return 5             │
│ trend_score < 15  → 短路返回                       │
│ 强度=0 → compute_total_score 跳过所有后续计算       │
└───────────────────────────────────────────────────┘
    │ 通过 ✓
    ▼
┌─ 第二层：洗盘质量条件激活 ───────────────────────┐
│ if stock_strength < 20: washout_quality = 0        │
│ "没有上涨前科的缩量下跌 = 阴跌，不是洗盘"           │
│ 堵死"缩量阴跌伪装洗盘"的跨维度套利路径              │
└───────────────────────────────────────────────────┘
    │
    ▼
┌─ 第三层：Sigmoid 门控乘法（第 720 行）────────────┐
│ gate = 1 / (1 + e^(-0.15 × (strength - 35)))      │
│                                                    │
│ strength=0:  gate≈0.005 → 总分×0.005 → 归零       │
│ strength=5:  gate≈0.011 → 总分×0.011 → 几乎归零    │
│ strength=20: gate≈0.095 → 总分×0.095 → 压到1/10   │
│ strength=35: gate≈0.500 → 总分×0.500 → 腰斩       │
│ strength=50: gate≈0.905 → 总分×0.905 → 轻微折扣    │
│ strength=80: gate≈0.999 → 总分×0.999 → 几乎不变    │
└───────────────────────────────────────────────────┘
    │
    ▼
┌─ 第四层：趋势一致性校验（第 648-676 行）──────────┐
│ 前期涨 + 近期回调 → consistency=100 → mult=1.00   │
│ 前期跌 + 近期跌   → consistency=0   → mult=0.50   │
│ 前期涨 + 近期涨   → consistency=30  → mult=0.65   │
└───────────────────────────────────────────────────┘
    │
    ▼
  total = raw × gate × consistency_mult
```

### 4.2 效果对比

| 股票状态 | 强度 | 洗盘 | 其他 | raw | gate | consist | **最终** |
|---------|------|------|------|-----|------|---------|---------|
| 强势+洗盘健康 | 80 | 75 | 70 | 74 | 0.999 | 1.00 | **73.5** |
| 中等+洗盘一般 | 45 | 50 | 55 | 51 | 0.818 | 0.75 | **31.3** |
| 弱势+缩量阴跌 | 5 | 0 | 50 | 29 | 0.011 | 0.50 | **0.2** |
| 微涨+疑似洗盘 | 25 | 55 | 45 | 38 | 0.182 | 0.75 | **5.2** |
| 下跌趋势 | 0 | 0 | 0 | 0 | — | — | **0.0** |

> 弱势股从旧架构的 38~42 分被压到接近 0 分，强势股几乎不受影响。

### 4.3 `compute_total_score` 执行顺序（第 678-730 行）

```python
def compute_total_score(self):
    # 1. 先算股票强度（电源开关，必须最先）
    stock_strength = self.score_stock_strength()

    # 2. 强度=0 → 直接出局，全维度归零，不浪费算力
    if stock_strength == 0:
        return all zeros

    # 3. 洗盘质量：传入强度做前置门控（<20 → 直接 return 0）
    washout = self.score_washout_quality(strength_score=stock_strength)

    # 4. 其余维度
    ...

    # 5. 原始线性加权（保留参考）
    raw = stock_strength×0.30 + washout×0.15 + ...

    # 6. Sigmoid 门控  ← P0 核心
    gate = 1.0 / (1.0 + exp(-0.15 × (stock_strength - 35)))

    # 7. 趋势一致性因子  ← P2
    consistency_mult = 0.5 + 0.5 × consistency

    # 8. 最终总分
    total = raw × gate × consistency_mult
```

### 4.4 Sigmoid 门控参数

| 参数 | 位置(行) | 默认值 | 含义 |
|------|----------|--------|------|
| 中心点 (center) | 720 | **35** | strength=35 时 gate≈0.5 |
| 陡峭度 (k) | 720 | **0.15** | 越大过渡越陡，越小越平缓 |
| 调高 center | 720 | — | 更严格：需要更强趋势才能拿到高 gate |
| 调低 center | 720 | — | 更宽松：中等强度也能通过 |
| 调高 k | 720 | — | 门控更陡，开关更"硬" |
| 调低 k | 720 | — | 门控更平，过渡更平滑 |

### 4.5 趋势一致性 `trend_consistency`（第 648-676 行）⚠️ P2 新增

**度量什么**：前期趋势方向 vs 近期调整方向是否逻辑一致。

| 场景 | consistency | multiplier | 含义 |
|------|:----------:|:----------:|------|
| 前期涨 + 近期回调 | 100 | ×1.00 | 健康洗盘，不打折 |
| 前期涨 + 近期也涨 | 30 | ×0.65 | 没有洗盘，打折 |
| 前期跌 + 近期也跌 | 0 | ×0.50 | 主跌浪，腰斩 |
| 其他 | 50 | ×0.75 | 中性 |

```python
def trend_consistency(self):
    pre_trend = np.sign(前期对数斜率)    # +1涨 / -1跌
    recent_mean = 近15日均价
    pre_end = 前期收盘价
    
    if pre_trend > 0 and recent_mean < pre_end * 0.98:
        return 100   # 前期涨，近期回调 → 健康洗盘
    elif pre_trend > 0 and recent_mean >= pre_end:
        return 30    # 前期涨，近期也涨 → 没有洗盘
    elif pre_trend < 0 and recent_mean < pre_end:
        return 0     # 前期跌，近期也跌 → 主跌浪
    else:
        return 50    # 中性
```
